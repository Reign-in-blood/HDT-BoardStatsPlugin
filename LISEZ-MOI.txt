BOARDSTATSPLUGIN v0.2.8
========================

NOUVELLE FONCTION : LAST COMBAT
-------------------------------
Pendant le combat, le plugin mémorise chaque serviteur qui apparaît sur le
plateau du joueur et de l'adversaire.

Pour chaque serviteur, il conserve uniquement :

- son attaque maximale observée ;
- ses points de vie maximaux observés.

Une même entité n'est jamais additionnée plusieurs fois lorsqu'elle reçoit
un bonus.

Exemple :

10/10 -> bonus +2/+2 -> 12/12

Contribution au bilan :

12 ATK / 12 HP

et non :

10/10 + 12/12 = 22/22


DÉGÂTS ET DIMINUTIONS
---------------------
Les dégâts ne réduisent pas le maximum enregistré.

Le plugin utilise GameTag.HEALTH, qui représente les points de vie maximaux
actuels. Les dégâts sont stockés séparément par Hearthstone.

Une baisse d'attaque ou de vie ne remplace jamais une valeur maximale déjà
enregistrée.


UNITÉS COMPTÉES
---------------
Chaque identifiant d'entité différent compte comme une unité :

- les serviteurs présents au début du combat ;
- les serviteurs invoqués pendant le combat ;
- les serviteurs arrivant depuis la main ;
- les tokens ;
- les nouvelles entités créées par Réincarnation.

Une augmentation des statistiques d'une entité existante ne crée pas une
nouvelle unité.


AFFICHAGE
---------
Pendant le combat :

- statistiques actuelles du joueur ;
- statistiques actuelles de l'adversaire.

Pendant le recrutement suivant :

- statistiques actuelles du plateau du joueur ;
- second cadre « LAST COMBAT » avec :

  PLAYER    UNITS / ATK / HP
  OPPONENT  UNITS / ATK / HP


COMPILATION
-----------
Cette version utilise également HearthDb.dll afin d'accéder aux points de
vie maximaux indépendamment des dégâts.

Lance normalement :

build_only.bat

Le script récupère automatiquement HearthstoneDeckTracker.exe et
HearthDb.dll dans l'installation de HDT.

La DLL finale est créée dans :

dist\BoardStatsPlugin.dll


CORRECTION v0.2.1 — COMPTAGE DES UNITÉS
----------------------------------------
La version précédente comptait les identifiants d'entités uniques. Cette
méthode n'était pas suffisante, car une même entité peut quitter le plateau
puis revenir pendant un combat.

La nouvelle version compte les apparitions dans la zone PLAY :

- première présence sur le plateau : +1 unité ;
- bonus de statistiques sans quitter le plateau : aucune nouvelle unité ;
- mort ou sortie du plateau : l'apparition est fermée ;
- retour ultérieur de la même entité : +1 nouvelle unité.

Le balayage utilise directement Core.Game.Entities et les tags de zone.
Il n'utilise plus Player.Minions pour le bilan « LAST COMBAT », car cette
collection masque temporairement les entités qui ont des changements de tags
en attente. Ces disparitions temporaires pouvaient rendre le comptage
incohérent.

Pour chaque apparition, seules les valeurs maximales sont retenues :

10/10 -> 12/12 -> 12/4 = contribution finale 12/12.


CORRECTION v0.2.2 — DOUBLE COMPTAGE DU JOUEUR
----------------------------------------------
Battlegrounds conserve les entités du plateau de recrutement du joueur tout
en créant des copies séparées pour le combat.

La version précédente pouvait donc voir simultanément :

- les 7 serviteurs persistants du recrutement ;
- les 7 copies réellement utilisées pendant le combat.

Résultat possible : 14 unités alors que 7 seulement avaient combattu.

La version 0.2.2 mémorise les identifiants et les positions du plateau pendant
le recrutement. Au début du combat, elle détecte les copies de combat puis
ignore définitivement les anciennes entités de recrutement jusqu'à la fin du
combat.

Même lorsque toutes les copies de combat sont mortes, le plugin ne revient
plus vers les entités persistantes du recrutement.

Une seconde protection évite qu'une disparition temporaire liée aux changements
de tags HDT soit interprétée comme une mort suivie d'une nouvelle invocation.


CORRECTION v0.2.3 — INITIALISATION DU COMBAT
---------------------------------------------
Le test « 1 serviteur, aucune invocation, résultat 2 unités » a montré que
le suivi par sorties et retours dans la zone PLAY n'était pas fiable.

La nouvelle méthode :

1. mémorise le plateau de recrutement ;
2. attend que le plateau visible du combat soit stable pendant plusieurs
   actualisations consécutives ;
3. enregistre ce plateau une seule fois ;
4. compte ensuite uniquement les nouveaux Entity.Id.

Une entité qui disparaît temporairement puis revient avec le même Entity.Id
reste donc une seule unité.

Les entités persistantes du recrutement sont exclues dès que les copies de
combat sont détectées.

Exemple attendu :

- 1 serviteur au début ;
- aucune invocation ;
- résultat : UNITS 1.

Les bonus restent traités comme auparavant :

10/10 -> 12/12 -> dégâts = contribution maximale 12/12.


CORRECTION v0.2.4 — UNITÉS LOGIQUES ET ALIAS
---------------------------------------------
Le plateau de recrutement est maintenant la référence du nombre initial.
Chaque serviteur initial crée exactement une unité logique.

Quand Hearthstone crée une nouvelle entité pour représenter ce même serviteur
durant le combat, le plugin associe le nouvel Entity.Id à l'unité logique
existante au lieu d'ajouter une unité. La correspondance utilise la position
sur le plateau et le CardId.

Seules les entités qui ne correspondent à aucune unité initiale sont comptées
comme de véritables invocations supplémentaires.

Un fichier BoardStatsPlugin_debug.log est créé à côté de la DLL. Il contient :
- les unités initiales ;
- les associations entre copies de combat et unités initiales ;
- les entités comptées comme invocations ;
- le total final.


MODIFICATION v0.2.5 — ORDRE D'AFFICHAGE
---------------------------------------
Les valeurs sont maintenant affichées avant leur libellé.

Avant :
ATK 256
HP 410
UNITS 8

Maintenant :
256 ATK
410 HP
8 UNITS

La disposition conserve des colonnes fixes et les valeurs numériques restent
alignées à droite.


MODIFICATIONS v0.2.6 — TYPOGRAPHIE ET LIBELLÉS
-----------------------------------------------
- Les chiffres utilisent maintenant la même police que les textes : Segoe UI.
- Un espace visuel fixe de 5 pixels est ajouté entre la valeur et ATK, HP ou UNITS.
- PLAYER est remplacé par YOU.
- OPPONENT est remplacé par FOE.
- YOU et FOE ont tous les deux trois caractères.
- La colonne réservée au nom est réduite pour éviter un grand espace vide.

Exemples :

YOU      256 ATK      410 HP
FOE      184 ATK      295 HP

YOU        8 UNITS    1250 ATK    1840 HP
FOE       10 UNITS    1470 ATK    2110 HP


MODIFICATIONS v0.2.7 — BANDEAUX DE TITRE
-----------------------------------------
Les deux blocs de contenu ont maintenant exactement la même largeur : 360 px.

Chaque ensemble est divisé en deux éléments noirs séparés :

1. un bandeau de titre de 20 px de haut ;
2. un espace de 2 px ;
3. le bloc contenant les statistiques.

Titre du premier bloc :

- pendant le recrutement : BOARD STATS en vert ;
- pendant le combat : PLAYER en vert, VS en blanc, OPPONENT en rouge.

Titre du second bloc :

- PREVIOUS COMBAT en blanc.

Le second titre et son contenu sont affichés uniquement pendant le recrutement
lorsqu'un résumé du combat précédent est disponible.


MODIFICATIONS v0.2.8 — SUPPRESSION DES NOMS DE LIGNE
-----------------------------------------------------
- YOU et FOE sont supprimés des deux blocs.
- Les couleurs verte et rouge distinguent les lignes.
- La largeur commune passe de 360 px à 300 px.
- Le titre de combat devient : PLAYER  VS  OPPONENT.
